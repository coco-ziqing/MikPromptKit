## 开发经验总结

### DO ✅
1. **先用sqlite3确认表结构再写API** — 避免Row.get() vs dict[]
2. **读写表统一** — 左栏v4读/移动旧表写→数据不一致
3. **Python脚本替换JS** — 避开复杂字符串嵌套语法错误
4. **内联style双重保险** — CSS !important + JS直接注入
5. **fixed定位独立挂载** — 按钮不在父元素内，避开transform影响

### DON'T ❌
1. 避免在Python -c中放复杂多行引号嵌套(用脚本文件)
2. git checkout回退要谨慎，会丢失未提交修改
3. onerror含嵌套引号在JS模板字符串中极容易出错
4. 两张表同时统计会重复计数(shadowed by shared IDs)
