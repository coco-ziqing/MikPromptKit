## Phase 8.5: 版本管理系统 (tag: v4.0.0-phase8.5-vm)

### 问题诊断
- `cards.py` update_card 编辑时不自动存档
- rollback 端点只恢复 content/meaning 两个字段（共6个字段）
- 旧 versions API 仅操作旧 prompts 表，v4 prompt_cards 脱节

### 修复内容
1. **编辑自动存档** (`cards.py` update_card): 每次编辑前将当前完整状态存入 `prompt_versions`，版本号自动+1
2. **完整回滚** (`cards.py` rollback): 回滚前存档当前版本，恢复全部6字段(content/meaning/scene/module/category)
3. **v4 版本历史** (`cards.py` GET /cards/{id}/versions): 返回当前+全量版本列表
4. **v4 版本 diff** (`cards.py` GET /cards/{id}/versions/diff/{v1}/{v2}): 5字段差异对比

### 测试: 5/5 通过

## Phase 9: 组装器引擎升级 (tag: v4.0.0-phase9-assembler)

### 问题诊断
三个组装器并存 (seedance.py v1 / composer_v3.py / seedance_v2.py)，核心引擎在 seedance_v2.py 但功能不足：
- 仅1种输出格式
- 分辨率模糊("4K")
- 无音频字段
- 无密度控制
- 时长硬限制15s

### 升级内容
1. **多格式引擎** `_make_structured_description()`: Seedance/Kling/MiniMax/ComfyUI/Raw 5种格式
2. **像素分辨率** `_calc_pixel_res()`: 画幅×分辨率→精确像素 (16:9 4K→3840×2160, 9:16→2160×3840)
3. **3档密度** compact/standard/detailed
4. **音频支持**: BGM+音效+对白字段，数据库迁移添加3列
5. **最大时长** 15s→60s
6. **镜头级负词**
