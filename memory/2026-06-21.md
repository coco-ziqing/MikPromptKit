  → retry 延迟调用也崩溃

  ↓
Layer 5: _escape 可能在 renderSidebar 调用时未定义
  ↓
Layer 6: signal_lights.js 在 app_core 之前执行，直接引用 App → ReferenceError
  ↓
Layer 7: renderSidebar 无 try-catch，任意中间异常 → 侧边栏完全空白

## 经验教训

### 1. `const` 声明顶层对象是危险的
跨 `<script>` 标签共享对象必须用 `var` 或显式 `window.App = ...`。`const` 是块级作用域，不设置 `window` 属性。

### 2. `self === this` 引用陷阱
`self.prop !== App.prop` 在 `self === App` 时永远为假。应使用命名函数表达式独立捕获引用。

### 3. IIFE 等待条件要用 try-catch
`if (!window.App)` 在 `App` 未定义时可能抛 ReferenceError（取决于上下文）。安全写法：
```javascript
(function check() {
    try { if (typeof App !== 'undefined' && App.fetchJSON) { ... } }
    catch(e) { setTimeout(check, 200); }
})();
```

### 4. `'use strict'` + `arguments.callee` 不兼容
严格模式下 `arguments.callee` 抛 TypeError。重试循环必须用命名函数。

### 5. 模块加载顺序决定生死
被依赖者必须在依赖者之前加载。`wc_bridge.js` 定义 `loadGroupTree`，`app_core.js` 调用它 — wc_bridge 必须排前面。

### 6. 浏览器缓存 + 版本号 = 双刃剑
修复迭代时需频繁升级版本号（`v=8→9.0→9.1→...→9.6→10.0`），否则浏览器坚持加载旧 JS。

### 7. 诊断面板比 F12 Console 快
排查阶段可在页面底部注入可见的 `#_wcDebug` 面板，绕过控制台过滤。

### 8. try-catch 是所有动态渲染函数的标配
200行的 `renderSidebar` 应从一开始就包裹 try-catch，否则任意中间异常导致整个侧边栏空白。

## 技术要点

### ComfyUI 桌面版特点
- 官方桌面版端口不固定（8000/8188/8743 等）
- 服务名为 `ComfyUI`（任务管理器可见）
- 默认绑定 `127.0.0.1`（仅本机可访问）

### 服务管理
- 杀进程：`Get-Process python \| Stop-Process -Force`
- 清残留连接：等待 TIME_WAIT 自然过期（~60秒）即可重新绑定端口
- 启动：`start-process -WindowStyle Hidden` 在 PowerShell 中启动后台进程
