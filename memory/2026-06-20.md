# 2026-06-20 Session Memory

## Summary
Bug-fixing session for PromptKit v4.1.0-phase13. Addressed multiple frontend regressions after Phase13 merges (P13.6).

## Completed Fixes

### 1. Version Display (`vv4.1.0` → `v4.1.0`)
- **File**: `app_core.js`
- **Change**: `v.replace('v','')` → `v.replace(/^v+/i, '')` — previous code only stripped first 'v', resulting in double 'v' prefix
- **Root cause**: `_syncVersion` overrides `brandVersion` in `app_editor.js` by splitting at `-` and prepending `'v' +`, while the original string already had a `v` prefix

### 2. Brand Name Unification
- Changed all instances to "咪卡Mik词库":
  - `index.html` `<title>` tag
  - `index.html` `div.brand` text
  - `app_core.js` `document.title` assignments

### 3. Sidebar Visibility in Word Card Manager
- **File**: `app_core.js`
- **Change**: `_hideSidebar()` → `_showSidebar() + _collapseSidebar()` — sidebar was completely hidden in wcmanager view instead of being collapsed (hamburger-accessible)

### 4. Media Assets View (v4_library) Restored
- **File**: `index.html`
- **Root cause**: During P13.6 merge, the `viewV4media` HTML block and `v4_library.js` script tag were accidentally removed
- **Fix**: Re-added both blocks

### 5. Health Check Frequency
- **File**: `app_core.js`
- **Change**: Added `sessionStorage._pk_health_checked` flag so health check runs only once per browser session (on first load after server restart), not on every F5 refresh

### 6. i18n Engine Rewrite
- **File**: `app_i18n.js` (rewritten), `en.json` (supplemented)
- **Change**: Replaced scattered `App._t()` calls with `data-i18n` DOM attribute approach via `_applyI18n()` scanner
- **Lang button**: Added directly in `index.html` header

### 7. Edit Mode Module Creation API
- **File**: `app_editor.js`
- **Change**: `createCustomModule()` switched from `GET /api/modules` POST to `POST /api/v4/word-cards/groups`
- **API format**: Backend returns `{ok:true, id:N}` — validated working
- **Also**: Switched from `this.fetchJSON` to native `fetch()` because `fetchJSON` silently returns `null` on non-200

### 8. wc_bridge loadModules Empty Groups
- **File**: `wc_bridge.js`
- **Change**: Added `?include_empty=true` to `/api/v4/word-cards/groups` request so newly created groups with 0 cards appear in sidebar

### 9. Sidebar Refresh After Create/Save
- **File**: `word_editor.js`, `app_editor.js`
- **Change**: Both `_createModule` (word_editor.js line 303) and save operations now call `await App.loadModules()` to refresh sidebar counts

### 10. app_editor.js Corruption Recovery
- **Root cause**: Python-based text replacements caused emoji encoding corruption — 5 `(function()` blocks with only 1 proper close
- **Recovery**: Restored from commit `06c47ca`, then re-applied the `createCustomModule` API fix using Node.js (preserves UTF-8 emoji)
