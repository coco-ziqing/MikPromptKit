## 关键技术决策

### DB 锁问题三类根因 → 统一修复模式
1. **健康检测** → 独立 `sqlite3.connect`（不共用 `get_db()`）
2. **safe_commit** → 3次重试 + 递增等待(50ms/150ms/400ms)
3. **翻译端点** → 三段短连接（读→close→Ollama→独立连接写缓存）

### 猴补丁链缺陷（长期风险）
4 个 composer 模块各自猴补丁 `switchView`，链式嵌套：
`atom_editor → scene_composer → unified_composer → character_composer → 原始 app_core.js`

**缓解**：`app_core.js` 新增统一 `else if` 分支做 `_hideSearchBox + _showSidebar + _collapseSidebar + 导航高亮`，猴补丁变为增强非唯一通道。
