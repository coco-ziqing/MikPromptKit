## 二、组装器时间轴逐段对齐（Phase 9.3，v69→v66，8次迭代）

### 最终方案：v9.3.9 — JS实测像素

**原理**：渲染后JS用 `getBoundingClientRect()` 读取分段实际像素位置，以绝对px写入刻度行。

```
1. 渲染两条独立flex行（相同flex-grow比例）
   [刻度行] transparent border补偿宽度差
   [分段行] 1px border

2. requestAnimationFrame → _syncTicks()
   读取seg[i].getBoundingClientRect().right → 分段右缘像素
   写入刻度行: <span style="position:absolute;left:Npx">N s</span>

3. 刻度行z-index:2, 分段行z-index:1, appendChild(保留flex-cell占位)
```

### 迭代历程
| 版本 | 方案 | 失败原因 |
|------|------|---------|
| v9.3 | 绝对定位 left/width | CSS两个百分比独立舍入→1px缝隙 |
| v9.3.1 | flex-grow分段 | overflow:hidden裁刻度 |
| v9.3.4 | 双flex独立行 | CSS space-between退化 |
| v9.3.5 | cell内abs left/right | 双容器子像素差异 |
| v9.3.7 | 刻度嵌入分段 | 用户要求上下两行 |
| v9.3.8 | 刻度叠入bar内 | left%与flex像素偏差 |
| v9.3.9 | **JS实测像素** | ✅ 最终方案 |

### 核心方法
- `_buildTimelineHTML()`：两行独立flex HTML
- `_syncTicks()`：`getBoundingClientRect()` 测像素→绝对定位刻度
- `_refreshTimeline()`：`requestAnimationFrame`→`_syncTicks()`
- `_rebindTimelineDrag()`：每次build后重新绑定拖拽

### 审阅弹窗同步
- `_renderSrTimeline()`：重写为flex-grow+刻度行
- `_syncSrTicks()`：测量sr分段像素→写入刻度

## 四、技术总结

### 教训：CSS百分比对齐不可靠
- 两个独立CSS属性(width vs left)的百分比→像素舍入不一致
- 两个独立flex容器的flex-grow分配存在子像素差异
- **结论**：精确对齐需求应走JS实测像素方案

### 最终可靠架构
```
刻度行 (position:relative, flex, transparent border)
  <span class="tick" style="position:absolute;left:Npx"> ← JS测量写入
分段行 (position:relative, flex, 1px border, z-index:1)
  <div style="flex-grow:2">镜头1</div>
  <div style="flex-grow:4">镜头2</div>
```

- 刻度被颜色条遮挡（已修复：appendChild保留flex-cell + z-index分层）
