## 二、组装器时间轴逐段对齐优化（Phase 9.3）

### 目标
- 时间轴刻度数值与分段断点精确对齐
- 分段间无缝隙邻接
- 修改镜头时长后实时刷新

### 迭代历程（9次尝试）

#### v9.3 — 绝对定位分段
- 分段改用 `position:absolute` + `left`/`width` 百分比
- 刻度收集所有分段边界点生成
- **问题**：两个 CSS 属性（width vs left）浮点→像素舍入不一致，段间出现1px缝隙

#### v9.3.1 — flexbox 分段
- 分段改用 `flex-grow: duration`，flex 引擎保证无缝邻接
- 刻度叠入 bar 内部 `position:absolute` + `left:X%`
- **问题**：`overflow:hidden` 裁剪了 `top:-20px` 刻度层

#### v9.3.4 — 两个独立 flex 行
- 刻度行和分段行各为独立 flex 容器，相同 flex-grow 比例
- CSS `justify-content:space-between` 放置右标签
- **问题**：space-between 在单元素 cell 中退化为 flex-start → 标签挤到左缘

#### v9.3.5 — cell 内绝对定位
- 刻度 cell 用 `position:relative`，标签 `position:absolute` left/right
- **问题**：两个独立 flex 容器，浏览器子像素舍入差异 → 累积错位

#### v9.3.7 — 刻度嵌入分段内
- 刻度放入分段 div 内，`position:absolute` 相对分段定位
- **问题**：用户要求必须是上下两行，不能放同一行

#### v9.3.8 — 刻度叠回 bar 内
- 刻度 `position:absolute` 挂在 bar 上，同容器基准
- `overflow:visible` + 分段自控圆角
- **问题**：`left:X%` 与 flex 子项实际像素仍有偏差

#### v9.3.9 — JS 实测像素（当前方案）
- 渲染后用 `getBoundingClientRect()` 读取分段实际像素位置
- 以绝对 px 值写入刻度行
- 新增 `_syncTicks()` 方法
- `_refreshTimeline()` 通过 `requestAnimationFrame` 触发同步

### v9.3.9 核心实现

**`_buildTimelineHTML()`**：生成两行独立 flex HTML
- 刻度行：`s2-timeline-ticks`（flex, transparent border补偿宽度差）
- 分段行：`s2-timeline-bar`（flex, 同 flex-grow 比例）

**`_syncTicks()`**：读取分段实际像素位置
```
bar.getBoundingClientRect() → 基准
seg[i].getBoundingClientRect() → 分段右缘像素
刻度行 innerHTML = <span style="position:absolute;left:Npx">
```

**触发点**：`_refreshTimeline()` → `requestAnimationFrame` → `_syncTicks()`
