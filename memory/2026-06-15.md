## 遇到的 6 个关键问题及修复

### 1. ZIP 路径分隔符问题 → 目录结构全毁
- **现象**：Windows 用 PowerShell `Compress-Archive` 打包 → 反斜杠 `\` → macOS 解压后 `backend\main.py` 变成扁平文件名，非文件夹
- **修复**：改用 Python `zipfile` + `as_posix()` 正斜杠路径
- **规则**：跨平台 ZIP 必须用 Python zipfile

### 2. macOS 隔离标记 + 执行权限
- **现象**：双击 `.command` 报 "没有正确的访问权限"
- **原因**：① `chmod` 未加执行权限 ② macOS Gatekeeper `com.apple.quarantine` 标记
- **修复**：`chmod +x` + `xattr -cr` 两步

### 3. `requirements.txt` 缺包
- **现象**：`RuntimeError: Form data requires python-multipart`
- **修复**：加 `python-multipart==0.0.9`
- **规则**：requirements.txt 必须完整，FastAPI UploadFile 依赖此包

### 4. NumPy 2.x 兼容性崩溃
- **现象**：`_ARRAY_API not found` + `NumPy 2.4.6 cannot run with modules compiled for NumPy 1.x`
- **根因**：macOS 老 PyTorch (2.2.2) 编译时链接 NumPy 1.x，与 NumPy 2.x 二进制不兼容
- **修复**：`numpy<2`
- **规则**：macOS 非 GPU 环境必须 `numpy<2`

### 5. semantic.py 异常捕获过窄
- **现象**：`NameError: name 'nn' is not defined` 崩溃，但 `except ImportError` 没兜住
- **根因**：sentence-transformers 导入链深处报 `NameError`，不是 `ImportError`
- **修复**：`except ImportError` → `except Exception`
- **规则**：ML 模块导入必须 `except Exception` 宽捕获

### 6. msvcrt Windows 专有模块
- **现象**：`main.py` 中有 `import msvcrt; msvcrt.getch()`
- **修复**：已在 try/except ImportError 中包裹 + `input()` fallback（更早前修复）

## 经验总结

> **一句话：跨平台封装的敌人是"路径分隔符"和"隐式版本假设"。**
> - 路径：永远用正斜杠 `/`
> - 依赖：永远锁 `numpy<2`，永远补全 `python-multipart`
> - 异常：永远 `except Exception`，永不用 `except ImportError` 兜 ML
> - 权限：macOS 永远 `chmod +x` + `xattr -cr`
> - 降级：非核心功能（视频/语义搜）不可阻断核心服务
