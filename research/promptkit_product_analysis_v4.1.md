## 1. 产品全景分析

### 1.1 产品定位

**PromptKit（咪卡MiK提示词助手）** 是一款面向**AI视觉创作者**（Midjourney/Stable Diffusion/Kling/Sora/Seedance使用者）的本地化、全功能提示词管理与AI视频组装工作站。

**一句话定位：** 局域网内可跨设备访问的AI创作者提示词一体化工作站。

**目标用户画像：**
- **AI绘画/视频创作者** — 需要管理大量提示词模板，跨平台（Midjourney、SD、Flux、Kling、Seedance等）使用
- **AI视频导演** — 需要多镜头结构化提示词组装，支持Seedance/Kling/Veo/Sora等平台格式输出
- **提示词设计师** — 需要精细化管理词库维度，并有版本管理、协作需求
- **工作流工程师** — 需要ComfyUI集成、批量处理、AI辅助优化

### 1.2 产品核心理念

| 维度 | 描述 |
|------|------|
| **数据主权** | 全部数据本地存储（SQLite），不上云、无需注册 |
| **多终端** | WebUI架构，同一局域网内任意设备（PC/手机/平板）访问 |
| **零依赖启动** | Python + 浏览器即可运行，无需配置数据库服务器 |
| **AI原生** | 内置Ollama集成、语义搜索、AI翻译、智能推荐 |
| **多平台兼容** | 输出适配Midjourney/Seedance/Kling/Sora/ComfyUI等多种格式 |

### 1.3 产品演进路线（时间线）

```
v3.0.0 (基础) → v3.1.x (语义搜索+版本管理) → v3.2.x (AI Workflow) → 
v3.3.x (拖拽/OCR/Markdown) → v3.4.x (ComfyUI缩略图/批量AI生成) → 
v3.5.x (翻译/PNG导出) → v3.6.x (收藏夹/词包/数据同步.pkb) → 
v3.7.x~v3.10.x (版本管理/媒体资产/OCR/截图导入) → 
v4.0.0 (统一词卡/组装器/角色库/深色模式) → 
v4.1.0 (统一词卡管理/桥接组件/AI全栈升级)
```

### 2.2 后端技术要点

| 要点 | 详细说明 |
|------|---------|
| **框架** | Python 3.14 + FastAPI + Uvicorn（异步ASGI服务器） |
| **数据库** | SQLite3 WAL模式（高并发读友好），FTS5全文索引，30+张表 |
| **API设计** | 25个路由模块，200+个端点，RESTful风格统一前缀 |
| **CORS** | 全开放`allow_origins=["*"]`，支持局域网任意设备跨域请求 |
| **中间件** | HTTP请求记录中间件（自动记录到monitor模块） |
| **端口探测** | 自动检测8080~8089可用端口，避免端口冲突 |
| **数据迁移** | 启动时幂等迁移（`_migrate_v4`），安全ALTER TABLE补列 |
| **备份系统** | 自动每日备份 + .pkb完整打包（含媒体文件）+ 历史管理 |
| **语义搜索** | Sentence-Transformers + all-MiniLM-L6-v2，异步重建索引 |
| **AI引擎** | Ollama本地模型池（16个模型分级路由），LLM Rerank |
| **健康检查** | 9项启动自检 + 30s后台Ollama/ComfyUI心跳 + 底部信号灯 |
| **线程安全** | `threading.local`数据库连接 + 写重试（最多3次） + 忙等待5秒 |

### 2.3 前端技术要点

| 要点 | 详细说明 |
|------|---------|
| **架构** | 原生JS SPA（无框架依赖），Bootstrap 5 CDN |
| **模块分割** | 15个独立JS文件（按功能拆分的模块化SPA） |
| **CSS** | 自定义2,500+行CSS，CSS变量驱动双主题（浅色/深色） |

| **文件处理** | 拖拽导入（JSON/.pt/PNG）、剪贴板粘贴、OCR截图 |
| **状态持久化** | localStorage保存视图、主题、列数、最近项目等 |

### 3.2 各模块详细交互流程

#### 3.2.1 首页词库

**入口：** → 点击`首页`按钮

**交互流程：**
1. 加载左侧`sidebar`模块树（内置+自定义）
2. 默认选中第一个模块，加载该模块提示词列表
3. 显示分类标签`categoryTabs`，可筛选子分类
4. 卡片列表展示：缩略图/内容/释义/标签/收藏状态
5. 点击卡片 → 内容复制到剪贴板，记录使用历史
6. 支持搜索框实时检索（FTS5 + 模糊匹配）
7. 支持列数滑块调整（1-6列）
8. 编辑模式下：卡片显示编辑按钮、可移动模块、可删除
9. 批量模式下：勾选→批量复制/导出(TXT/MD/JSON/PNG/.pt)/加词包/AI缩略图

**数据流：**
```
GET /api/prompts?module=X&category=Y&page=N 
→ 后端FTS5全文搜索/模糊LIKE匹配 
→ 返回分页数据+收藏归属信息 
→ 前端渲染卡片网格
```

#### 3.2.2 提示词组装器（核心差异化功能）

**入口：** → 点击`提示词组装器`按钮

**子模块：**
- **项目管理** — 创建/编辑/删除/搜索项目
- **镜头管理** — 多镜头时间轴编排（拖拽排序/插入/复制/粘贴/导出）
- **字段填充** — 点击字段芯片打开词卡选取器（25+维度）
- **全局配置** — 总时长/画幅/分辨率/全局风格/转场/负词
- **音频编排** — 角色旁白/BGM/音效（镜头级+全局级）
- **角色管理** — 出演角色选择（关联角色库）
- **拓展单元** — 自定义字段分组管理
- **拼装引擎** — 5格式×3密度输出（compact/standard/detailed）
- **模板闭环** — 组装器内容回写场景模板（覆盖/新建副本）
- **智能导入** — 文本→反向解析→自动匹配词卡→创建项目
- **时间锁** — 锁定/解锁镜头时长+自动重算

**详细交互流程：**
```
1. 创建/选择项目
2. 全局配置面板：设置总时长、画幅、分辨率、风格、负词
3. 镜头列表（默认1个镜头，最长60s/最多30个镜头）
4. 每个镜头包含：
   a. 基础字段区：运镜/主体/场景/构图/光影（点击芯片弹出词库选取器）
   b. 拓展字段区：动作/焦段/质感/速率/情绪/调色/天气/视角等
   c. 音频字段区：角色旁白/BGM/音效（可禁用镜头音频）
   d. 角色字段区：选择出演角色
   e. 拓展单元区：自定义分组字段
5. 字段填充交互：
   点击芯片 → 后端GET /api/v4/word-cards/groups → 右面板渲染词库列表
   点击词条 → POST /api/v4/word-cards/picker/link → 字段内容更新
6. 时间轴管理：
   修改时长/锁定→后端重算引擎_recalculate_scene_times
   拖拽排序→POST reorder
7. 拼装输出：
   选择format(seedance/kling/minimax/comfyui/raw)
   选择density(compact/standard/detailed)
   可选include_audio(音频要素融入描述)
   → POST /api/seedance/v2/projects/{id}/compose
   → 返回text+json结构化输出
8. 模板闭环：
   点击"更新模板"→覆盖原模板/新建副本
9. 智能导入：
   粘贴纯文本提示词 → POST /api/seedance/v2/parse
   → 解析时间轴/字段 → 智能匹配词卡 → 创建项目+填充字段
```

#### 3.2.6 OCR截图导入

**交互流程：**
```
1. 点击截图导入按钮（顶部相机图标）

3. 前端显示预览图片
4. 用户确认后→POST /api/ocr/analyze
5. 后端处理：
   a. 保存原图到originals目录
   b. 生成缩略图
   c. Ollama OCR模型（llama3.2-vision）分析图片
   d. 提取文字、检测语言（中文/英文路由）
   e. 结构化解析→模块/分类/标签/内容/释义
6. 返回值供用户确认→自动写入数据库
```

#### 3.2.7 数据同步.pkb

**交互流程：**
```
1. 点击同步按钮→打开同步面板
2. 包列表：名称/大小/时间/含媒体标记

4. 操作：

   - 导出纯DB包 → POST /api/sync/export-no-media
   - 恢复包（自动备份当前数据）→ POST /api/sync/restore/{name}
   - 上传.pkb文件导入 → POST /api/sync/upload
   - 删除包 → DELETE /api/sync/packages/{name}
5. 自动清理：保留最近20个包
```

#### 3.2.8 服务监测仪表盘

**交互流程：**

```
2. 显示面板：

```

## 4. 竞品深度对比分析

### 4.3 竞争优势分析（SWOT）

#### 优势 (Strengths)

1. **本地数据主权** — 所有数据在本地SQLite，不上云、不需要账号注册，数据安全可控
2. **局域网多终端** — 手机/平板/电脑任意设备可访问，无需安装客户端
3. **全链路创作者工具** — 从词库管理→词卡编辑→多镜头编排→多格式输出的完整闭环
4. **零成本** — 完全免费开源，相对于$49~$249/月的竞品具有巨大价格优势
5. **AI视频组装独占差异化** — 目前市场上唯一独立的、支持5种AI视频平台格式的本地化组装工具
6. **丰富的种子数据** — 165条内置提示词+302条词库资产，开箱即用
7. **Ollama本地AI集成** — 无需调用外部API，完全本地化AI辅助
8. **深色主题+响应式** — 专业级UI体验，媲美商业产品的设计质量

#### 劣势 (Weaknesses)

1. **单用户无协作** — 不支持多账号、团队协作、权限管理
2. **无A/B测试** — 无法对不同prompt版本做A/B测试对比效果
3. **无Git版本控制** — 版本管理基于SQLite，不支持Git仓库集成
4. **无移动端原生App** — 虽然是响应式WebUI，但无原生App体验
5. **无API Key管理** — 不支持第三方LLM/SD的API key管理与轮换
6. **无Prompt Marketplace** — 无法买卖/分享提示词
7. **无CI/CD集成** — 不支持与DevOps流水线集成
8. **无Prompt的LLM效果追踪** — 无法追踪同一个prompt在LLM上输出的质量变化

#### 机会 (Opportunities)

1. **AI视频生成爆发期** — Seedance/Kling/Veo等平台的用户快速增长，需要独立的提示词管理工具
2. **数据隐私意识提升** — 越来越多的创作者关注数据主权，本地化方案受欢迎
3. **开源生态** — 可以借鉴Langfuse模式，通过开源吸引社区贡献
4. **插件系统** — 开放API和插件接口，让社区开发自定义扩展
5. **多语言/国际化** — 支持英文界面吸引海外用户
6. **企业版/付费版** — 团队协作、LDAP集成、审计日志等高级功能可商业化

#### 威胁 (Threats)

1. **AI视频平台自带工具** — Kling/Seedance等平台可能强化官方编辑器功能
2. **竞品开源化** — Langfuse等产品也支持自托管并持续迭代
3. **技术债务** — 前端4年积累15个JS文件，维护成本逐步升高
4. **Python依赖运维** — 用户需要自行安装Python和依赖，有门槛
5. **Ollama兼容性** — 依赖Ollama的版本兼容，断代风险
