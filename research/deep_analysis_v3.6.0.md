# PromptKit 深度能力分析 — 从数据全生命周期看升级方向

> 分析框架：以"AI提示词数据全生命周期"为主线，结合工具现有优势，推演高杠杆升级方向
> 分析日期: 2026-06-02

---

## 一、核心前提：PromptKit 的独特位置

要回答"还需要什么能力"，首先要理解当前工具在竞争格局中的**独特生态位**：

```
          【云协作型】                         【纯学习型】
     Braintrust / Agenta / Langfuse          Anki / Quizlet
          ↑                                       ↑
          │            PromptKit                   │
          │       (本地媒体+提示词工作站)             │
          ↓                                       ↓
     【纯CRUD工具】                          【企业DAM】
     prompt-manage / PromptPal             Adobe AEM / Brandfolder
```

**PromptKit 的独特性在于**：它是唯一同时具备 **提示词管理 + 媒体管理 + 本地离线 + 语义搜索** 的工具。

所以升级方向应该是：**放大现有的独特组合优势，而非盲目对标某个单一品类。**

## 三、高杠杆升级方向分析

### 🔴 方向A：构建"测试→分析→优化"闭环（最高杠杆）

这是当前最大的空白，也是最能放大现有价值的投入。

#### A1 精准记录提示词使用效果

```
当前   复制 → 计数+1 （极简）
┌─────────────────────────────────────┐
│ 升级后 复制→ 记录使用场景               │
│        → 记录使用的AI模型              │
│        → 记录输出质量评分(用户反馈)      │
│        → 记录token消耗(估算)           │
└─────────────────────────────────────┘
```

**为什么重要**：
- 177 条词条，使用计数仅 12 次——说明用户只是存了但很少用
- 不记录效果就无法判断提示词好坏
- 不记录场景就无法优化提示词

**实现路径**：
- 扩展 `usage_history` 表：增加 `model_used`、`rating`、`context_note`、`token_estimate` 字段
- 复制弹窗增加简短的"效果评分"（👍/👎 或 1-5分）
- 统计面板展示：哪些提示词效果好、哪些从不使用

#### A2 提示词质量评分体系

**核心思路**：给每条提示词一个综合质量分，指导用户优化。

```python
quality_score = (
    content_clarity     * 0.3 +  # 内容清晰度（长度+结构）
    usage_frequency     * 0.2 +  # 使用频率
    avg_user_rating     * 0.3 +  # 用户评分
    semantic_uniqueness * 0.1 +  # 语义独特性（vs其他词条）
    media_completeness  * 0.1    # 媒体完善度
)
```

**UI 表达**：
- 卡片上显示质量分徽章（🔥 85/100）
- 低分词条提示"此提示词使用次数少，考虑优化或删除"
- 高分词条自动置顶推荐

#### A3 AI辅助提示词优化（利用已有Ollama能力）

```
┌──────────────────────────────────────────────────┐
│  编辑弹窗 → "AI优化建议" 按钮                     │
│  → 调用Ollama分析：                                │
│    1. 清晰度评分 + 改进建议                        │
│    2. 提示词结构（指令/上下文/输出格式）完整性检查    │
│    3. 同类提示词对比（差异化建议）                   │
│    4. 一键应用优化版本                              │
└──────────────────────────────────────────────────┘
```

**Ollama 已就绪，无需新增依赖。**

### 🔴 方向B：让媒体资产"活"起来（乘法效应）

PromptKit 的媒体能力是独特优势，但目前媒体只是"配图/配视频"。真正的价值在于：

#### B1 图片→提示词反向提取（多模态能力）

```
┌─────────────────────────────────────────────┐
│  图片/视频上传时自动:                          │
│  1. OCR提取图中文字 → 建议作为提示词内容        │
│  2. 图片风格分析 → 建议标签/分类               │
│  3. 同类图片关联 → 推荐相关提示词              │
└─────────────────────────────────────────────┘
```

**实现**：已有 OCR 模块，可扩展为上传时自动触发。

#### B2 媒体为提示词的"效果证据链"

```
提示词 A ──使用──→ 生成图片 X ──关联──→ 保存为缩略图/原图
                       ↓
                   记录：提示词A → 出图X → 评分4/5
                       ↓
                   其他用户看到：这个提示词生成过这样的效果
```

**价值**：从"文字描述"到"可见效果"的闭环，极大降低选择提示词的认知成本。

#### B3 相似图片基于内容搜索（不依赖文字标签）

- 利用图片哈希/特征向量，查找"看起来像"的提示词
- 用户上传一张参考图 → 找到能生成类似效果的提示词

### 🟠 方向C：从"提示词仓库"到"提示词工作台"

目前工具是**仓库模式**：存→搜→复制。升级方向是**工作台模式**：创作→测试→评估→迭代。

#### C1 Prompt Playground 增强

已有的 Playground 是单次测试。增强为：

```
┌─────────────────────────────────────────────────┐
│  Playground 2.0                                  │
│  ┌─ 测试记录 ────────────────┐                   │
│  │  #1 2026-06-01 模型:gpt4  │  效果:★★★★       │
│  │  #2 2026-06-01 模型:claude│  效果:★★★        │
│  │  #3 2026-06-02 模型:gpt4  │  效果:★★★★★      │
│  └────────────────────────────┘                   │
│  [重新测试] [并排对比] [导出报告]                   │
└─────────────────────────────────────────────────┘
```

#### C2 提示词变体管理

```
主版本: "写一篇关于{主题}的文章"
   ├── v1: "写一篇关于{主题}的800字文章"
   ├── v2: "写一篇关于{主题}的学术风格文章"
   └── v3: "用{角色}的口吻写一篇关于{主题}的文章"
```

每个变体可独立测试、评分、标记最佳版本。

### 🟠 方向D：数据资产化——让存量词条产生更大价值

177 条提示词已经是一个知识库。升级方向是**让这些数据自我增值**。

#### D1 知识图谱化

```
当前: 扁平列表 + 分类筛选
升级: 关联网络视图

  构图 ──→ 光线构图
   │          │
   │          └──→ 黄金时段（相关提示词:3条）
   │                      │
   │                      └──→ 日出（相关提示词:5条）
   │
   └──→ 色彩搭配 ──→ 暖色调（相关提示词:7条）
```

**实现**：基于标签重合度+内容语义相似度，自动构建关联图。

#### D2 "今日推荐" 智能引擎

- 基于历史使用模式 + 间隔复习排程
- 每天推荐 5 条"该复习/该试试"的提示词
- 用户反馈 → 调整推荐权重

#### D3 导出体系的商业化增值

当前的 PNG 导出已很强大。增加：

| 导出格式 | 用途场景 |
|---------|---------|
| **PDF 手冊** | 打印/分享/交付客户 |
| **CSV 表格** | Excel分析/数据迁移 |
| **OpenAI 格式** | 直接导入GPTs |
| **可分享链接** | 局域网内发个链接给别人看 |

### 🟡 方向E：用户体验的"最后一公里"优化

#### E1 全键盘操作
- `j/k` 上下卡片导航
- `Enter` 进入/离开卡片详情
- `c` 复制当前卡片
- `e` 编辑当前卡片
- `/` 聚焦搜索框
- `Esc` 关闭弹窗
- `Ctrl+N` 新建词条

#### E2 拖拽成像工作流
- 从左边的图片库拖图片到提示词 → 自动关联
- 从提示词拖到导出区 → 直接生成导出文件

#### E3 批量导入实用化
- CSV 批量导入（带分类/标签/释义）
- Chrome 扩展一键保存（右键 → 保存到 PromptKit）
- URL 抓取（从网页自动提取提示词）

## 四、分层优先级建议

### Tier 0 — 立刻做（2天以内，解决核心断层）

```
1. 使用效果记录
   └─ 复制弹窗增加评分反馈 → 数据积累 → 统计可展示
   
2. 提示词质量评分
   └─ 基于使用频率+评分的综合分 → 帮助用户发现"废词条"
   
3. AI优化建议（利用现有Ollama）
   └─ 编辑弹窗增加"AI建议"按钮 → 最直观的"有用感"
```

### Tier 1 — 近期做（1周，构建闭环）

```
4. Playground 增强
   └─ 测试历史记录 + 效果对比

5. 提示词变体管理
   └─ 同一条提示词的多个版本对比

6. 媒体证据链
   └─ 图片/视频自动关联使用记录
```

### Tier 2 — 中期做（2-4周，放大优势）

```
7. 知识图谱
8. 今日推荐引擎
9. 全键盘操作
10. 导出格式扩展（PDF/CSV/OpenAI）
```

## 五、核心结论

**一句话：PromptKit 当前的问题是"存了很多，用得很少"。**

升级的核心目标不是增加更多存储/管理功能，而是围绕**"使用→分析→优化"闭环**构建能力，让已有的 177 条提示词产生实际的生产力价值。

**最有杠杆效应的三个动作：**
1. ✅ **复制时加评分** → 立刻产生使用数据
2. ✅ **AI优化建议** → 利用已有Ollama基础设施，零成本启动
3. ✅ **质量评分体系** → 让用户知道自己存的提示词哪些值得用、哪些该删

这三点加起来不超过 2 天开发量，但能从根本上改变用户与工具的交互模式——从"存储者"变为"使用者"。
