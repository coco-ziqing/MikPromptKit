# 标准工作流程（OpenClaw 调用 + 工具落地部署流程）
## 一、全新开发流程（从零搭建整套工具）
1. 需求确认：明确核心功能、端口规划、局域网访问要求、终端适配范围。
2. 架构设计：定义目录结构、数据表/数据结构、API接口、页面布局、网络监听规则。
3. 代码开发
   - 后端：实现数据存储、增删改查、检索接口，配置监听 `0.0.0.0` 支持局域网访问。
   - 前端：编写自适应WebUI页面，对接后端接口，调试多浏览器/移动端兼容性。
4. 本地全量测试
   - Windows本机功能测试：新增、检索、复制、存储校验。
   - 局域网测试：使用手机/其他电脑，通过内网IP+端口访问，验证全功能可用。
   - 稳定性测试：多终端同时访问，验证并发无异常。
5. 部署封装：编写启动脚本，使用PyInstaller打包为Windows EXE程序。
6. 文档输出：源码、程序包、部署步骤、局域网访问教程、排错文档。

## 二、迭代开发流程（基于资料/需求更新）
1. 研读用户提供的参考资料、新功能、优化需求、网络适配要求。
2. 分析现有代码，确定修改模块（前端界面、后端接口、网络配置、数据结构）。
3. 增量修改代码，保证原有基础功能、局域网访问能力不失效。
4. 分场景测试：本机测试 + 局域网跨设备测试，验证新功能与原有功能正常。
5. 重新打包（如需），同步更新所有配套文档与配置说明。
6. 交付更新包、修改说明、新版部署指引。

## 三、故障修复流程（运行/访问异常）
1. 收集问题：报错信息、运行环境、设备类型、访问地址、复现步骤。
2. 问题定位：区分「服务启动异常」「本机可访、局域网无法访问」「界面功能异常」「数据异常」四类问题。
3. 针对性修复代码、网络配置、防火墙规则或前端样式。
4. 复测验证，确保本机+局域网多终端均可正常使用。
5. 输出修复方案 + 修复后程序/代码。

## 四、工具自身运行&访问流程
1. Windows主机双击EXE/执行启动脚本 → 后端服务启动，监听指定端口与全部网卡。
2. 本机访问：浏览器打开 `http://127.0.0.1:端口`，进入WebUI操作。
3. 局域网其他设备：同一网络下，浏览器打开 `http://[主机内网IP]:端口`，跨平台使用。
4. 日常操作：录入提示词→保存→关键词检索→查看/编辑/复制词条。
5. 关闭程序即可停止服务，本地数据自动持久化保存。

## 五、OpenClaw 交互规则
1. 仅响应工具开发、功能迭代、代码调试、局域网部署、故障排查、程序打包类指令。
2. 收到参考资料优先研读，严格结合资料内容进行开发与优化。
3. 输出顺序：方案说明 → 完整代码/配置 → 部署步骤 → 局域网访问指引。
