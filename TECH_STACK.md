# 技术栈规范（Windows后端 + 局域网跨平台WebUI 专用）
## 整体架构
B/S 架构：Windows主机运行后端服务作为服务端，局域网内所有设备通过浏览器访问WebUI，纯内网运行，无公网依赖。

## 一、后端技术（Windows 服务核心）
1. 主体框架：Python + FastAPI，轻量高性能，Windows兼容性优秀，适配并发请求。
2. Web服务：Uvicorn，配置监听 `0.0.0.0`（监听全部网卡），实现局域网IP访问。
3. 数据存储
   - 首选：SQLite 嵌入式数据库，单文件持久化，免部署独立服务。
   - 备选：JSON 文件存储，适用于极简小型场景。
4. 依赖管理：统一输出 requirements.txt，一键安装所有依赖。
5. 网络配置：默认端口 8080，支持配置文件自定义端口；兼容Windows防火墙规则。

## 二、前端 WebUI 技术（跨平台浏览器适配）
1. 基础技术：HTML5 + CSS3 + 原生JavaScript，拒绝重型框架，保证多终端加载速度。
2. 样式框架：Bootstrap 5，原生自适应布局，兼容电脑、手机、平板全尺寸设备。
3. 交互逻辑：JS 对接后端API，实现全功能交互，适配移动端触控操作。
4. 访问地址规范
   - 本机访问：http://127.0.0.1:端口号
   - 局域网访问：http://[Windows主机内网IP]:端口号

## 三、Windows 打包 & 运维工具
1. 打包工具：PyInstaller，打包为独立EXE程序，支持单文件/目录包两种格式。
2. 系统适配：Windows 10 / Windows 11 全版本兼容，统一UTF-8编码，杜绝中文乱码。
3. 路径规范：全部使用**相对路径**，程序可放置在Windows任意目录运行。
4. 网络工具：依托Windows自带命令查看内网IP、排查端口占用、配置防火墙。

## 四、环境与访问约束
1. 编码：全局 UTF-8，统一中英文内容编码规则。
2. 权限：默认普通用户权限运行，如需放行防火墙端口，提供对应操作指引。
3. 网络：仅针对**局域网内网**设计，不做公网穿透、公网部署相关开发。
